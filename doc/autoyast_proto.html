YaST2 inetd module in autoyast environment

Source structures (XML) description

Whole configuration is a list with the following entries:

tag
 data type
 description
changed
 boolean If service has changed properties (like Status, Protocol, ...), this flag is set to true.
Note: This property is module internal. If you change some properties, this must be set to true.
comment
 string
 Comment for configuration line in inetd.conf (xinetd files)
created
 boolean
 If any service is added by user or installed from distribution rpms, this flag is set to true for this service.
Note: This property is module internal. If you specify new service, you must also add this flag.
enabled
 boolean
 If true, service will be activated.
group
 string
 See inetd (xinetd) man pages.
iid
 string
 Module internal unique identifier.
For currently installed service contains something like path to (x)inetd configuration file.
If service is added manually, this property contains the word `new' followed by unique number (last stored in last_created tag).
If service is installed from distribution rpm, this property contains the word `inst' followed by unique number too with the same rules.
max
 string
 See inetd (xinetd) man pages.
protocol
 string
 Service protocol specification (See inetd (xinetd) man pages for more details.)
server
 string
 Contains full path to service program.
If service is beeing installed, this property contains only server program filename (without path!) and (rpm-)package name in rounded brackets.
server_args
 string
 See inetd (xinetd) man pages.
service
 string
 Service name.
socket_type
 string
 See inetd (xinetd) man pages.
user
 string
 See inetd (xinetd) man pages.
wait
 boolean
 See inetd (xinetd) man pages.



last_created
 integer Contains last unique number for new iid generator (for manually added or installed from rpm).
netd_service
 symbol
 Contains `inetd' if inetd will be configured, or `xinetd' for xinetd.
netd_status
 integer
 The value 0 means the server ((x)inetd) will be started, any other value means the server will be stopped and disabled.
service_package
 list
 This list contains packages for internet super server (inetd or xinetd). You can install booth servers, but only one can be enabled at run-time and only one can be configured with this YaST2 module.
xinetd_packages
 list
 This list contains package names for services for xinetd. If you write your own configuration file for autoyast, you can skip filling this property. It is used only for `Configuration Tree' output.
