(x)inetd module in Autoyast in SuSE Linux 8.3

Authors:    Martin Vidner <mvidner@suse.cz> and Petr Hadraba <phadraba@suse.cz>

This document describes the new proposed format of the inetd/xinetd part of the Autoyast profile. It focuses on creating the profile without using the GUI Configuration Management System.

Source structures (XML) description

Example:
<profile>
 <configure>
  ...
  <inetd>
    <netd_service config:type="symbol">xinetd</netd_service>
    <netd_status config:type"symbol">start</netd_status>
    <netd_conf config:type"list">
      <listentry>
        <service>tftp</service>
        <protocol>udp</protocol>
        <server>/usr/sbin/in.tftpd</server>
        <script>tftp</script>
        <enabled config:type"boolean">true</enabled>
      </listentry>
      <listentry>
        ...
      </listentry>
      ...
    </netd_conf>
  </inetd>
  ...
 </configure>
</profile>

Description

xinetd

Each service is identified by key-words <service>, <protocol>, <server> and <script>. These entries must be always present. Other options (like <enabled>) can be missing; in this case, default values will be used.inetd.

inetd

The situation is the same as in xinetd. But only <service>, <protocol> and <server> are used.

Note

The flags <changed>, <created> and <deleted> will be generaded automaticaly.
